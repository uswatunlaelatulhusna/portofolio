Bab 6: Keamanan Cloud Computing

Perlindungan data, sistem, dan layanan pada lingkungan cloud

1. Pengertian Keamanan Cloud Computing

Keamanan cloud computing adalah serangkaian kebijakan, teknologi, dan kontrol yang dirancang untuk melindungi data, aplikasi, dan infrastruktur cloud dari ancaman dan serangan.

Keamanan cloud menjadi sangat penting karena data disimpan dan diakses melalui internet.

2. Tantangan Keamanan Cloud

  • Kebocoran data (Data Breach)
  • Akses tidak sah
  • Serangan siber
  • Ketergantungan pada penyedia cloud
  • Kesalahan konfigurasi sistem

3. Prinsip Dasar Keamanan Cloud

  • Confidentiality: Data hanya diakses pihak berwenang
  • Integrity: Data tidak berubah tanpa izin
  • Availability: Layanan selalu tersedia
Prinsip ini dikenal sebagai CIA Triad.

4. Mekanisme Keamanan Cloud

a. Enkripsi Data

Data dienkripsi saat disimpan (data at rest) dan saat dikirim (data in transit).

b. Autentikasi dan Otorisasi

  • Username & Password
  • Multi-Factor Authentication (MFA)
  • Role Based Access Control (RBAC)

c. Firewall dan Network Security

Firewall melindungi jaringan cloud dari akses tidak sah.

d. Backup dan Recovery

Data dicadangkan secara berkala untuk menghindari kehilangan data.

6. Ancaman Keamanan Cloud

Contoh ancaman: Malware, Phishing, DDoS, Account Hijacking
  • Malware: Program berbahaya
  • Phishing: Penipuan untuk mencuri data
  • DDoS: Serangan untuk melumpuhkan layanan

7. Tanggung Jawab Keamanan Cloud

Keamanan cloud menggunakan model Shared Responsibility.

  • Penyedia cloud: Infrastruktur & jaringan
  • Pengguna: Data, akses, dan konfigurasi

8. Kesimpulan Bab 6

Keamanan cloud computing merupakan aspek krusial yang harus direncanakan dengan baik agar data dan layanan tetap aman, andal, dan terpercaya.
