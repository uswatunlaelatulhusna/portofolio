Bab 2: Model Layanan Cloud Computing

Memahami jenis layanan cloud berdasarkan tingkat kontrol dan tanggung jawab pengguna.

1. Pengantar Model Layanan Cloud

Model layanan cloud computing menjelaskan bagaimana sumber daya komputasi disediakan kepada pengguna. Setiap model layanan memiliki tingkat kontrol, tanggung jawab, dan kemudahan penggunaan yang berbeda.

Semakin ke arah SaaS, pengguna semakin mudah menggunakan layanan, namun kontrol sistem semakin kecil.

2. Infrastructure as a Service (IaaS)

Infrastructure as a Service (IaaS) adalah model layanan cloud yang menyediakan infrastruktur dasar seperti server virtual, penyimpanan (storage), dan jaringan. Pada model ini, penyedia cloud hanya menyediakan infrastruktur, sedangkan pengguna bertanggung jawab atas sistem operasi dan aplikasi.

Karakteristik IaaS:

  • Kontrol penuh terhadap sistem operasi
  • Dapat mengatur konfigurasi server sendiri
  • Skalabilitas tinggi

Contoh Penggunaan:

  • Hosting website skala besar
  • Server aplikasi
  • Lingkungan pengujian (testing environment)
Contoh layanan IaaS: Amazon EC2, Google Compute Engine, Microsoft Azure Virtual Machine

3. Platform as a Service (PaaS)

Platform as a Service (PaaS) menyediakan platform lengkap untuk pengembangan aplikasi, termasuk sistem operasi, database, dan framework. Pengguna hanya fokus pada pengembangan aplikasi tanpa perlu mengelola server.

Karakteristik PaaS:

  • Tidak perlu mengelola server
  • Mendukung kolaborasi tim developer
  • Mempercepat proses deployment aplikasi

Contoh Penggunaan:

  • Pengembangan aplikasi web
  • API dan backend aplikasi
  • Prototype aplikasi
Contoh layanan PaaS: Google App Engine, Heroku, AWS Elastic Beanstalk

4. Software as a Service (SaaS)

Software as a Service (SaaS) adalah model layanan cloud di mana pengguna langsung menggunakan aplikasi yang sudah jadi melalui internet. Semua pengelolaan sistem dilakukan oleh penyedia layanan.

Karakteristik SaaS:

  • Tidak memerlukan instalasi
  • Dapat diakses melalui browser
  • Pembaruan dilakukan otomatis

Contoh Penggunaan:

  • Email dan dokumen online
  • Aplikasi meeting online
  • Sistem manajemen pembelajaran
Contoh layanan SaaS: Gmail, Google Docs, Zoom, Microsoft 365

5. Perbandingan Model Layanan Cloud

  • IaaS: Kontrol paling besar, tanggung jawab paling tinggi
  • PaaS: Seimbang antara kontrol dan kemudahan
  • SaaS: Paling mudah digunakan, kontrol paling kecil
Pemilihan model layanan cloud harus disesuaikan dengan kebutuhan, kemampuan teknis, dan tujuan penggunaan.
