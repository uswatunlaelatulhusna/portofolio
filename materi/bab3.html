Bab 3: Model Deployment Cloud Computing

Mengenal cara penerapan cloud computing berdasarkan kepemilikan dan akses pengguna.

1. Pengertian Model Deployment Cloud

Model deployment cloud computing menjelaskan bagaimana infrastruktur cloud diterapkan dan siapa saja yang dapat mengaksesnya. Pemilihan model deployment sangat berpengaruh terhadap keamanan, biaya, dan fleksibilitas sistem.

Model deployment menentukan siapa pemilik cloud dan siapa yang boleh menggunakannya.

2. Public Cloud

Public Cloud adalah model deployment cloud di mana layanan cloud disediakan oleh pihak ketiga dan digunakan oleh banyak pengguna (multi-tenant). Infrastruktur sepenuhnya dimiliki dan dikelola oleh penyedia layanan cloud.

Karakteristik Public Cloud:

  • Dapat diakses oleh umum melalui internet
  • Biaya relatif lebih murah
  • Skalabilitas sangat tinggi

Contoh Penggunaan:

  • Website publik
  • Aplikasi startup
  • Media penyimpanan online
Contoh Public Cloud: Amazon Web Services (AWS), Google Cloud Platform, Microsoft Azure

3. Private Cloud

Private Cloud adalah model cloud yang digunakan secara khusus oleh satu organisasi. Infrastruktur dapat berada di dalam data center organisasi atau dikelola oleh pihak ketiga.

Karakteristik Private Cloud:

  • Akses terbatas hanya untuk organisasi tertentu
  • Tingkat keamanan lebih tinggi
  • Kontrol penuh terhadap data

Contoh Penggunaan:

  • Sistem keuangan perusahaan
  • Data pemerintahan
  • Aplikasi internal perusahaan
Private Cloud cocok untuk organisasi dengan kebutuhan keamanan tinggi dan regulasi ketat.

4. Community Cloud

Community Cloud adalah cloud yang digunakan oleh beberapa organisasi yang memiliki kepentingan dan kebutuhan yang sama, seperti regulasi, keamanan, atau tujuan bisnis tertentu.

Karakteristik Community Cloud:

  • Digunakan oleh komunitas tertentu
  • Biaya ditanggung bersama
  • Keamanan disesuaikan kebutuhan komunitas

Contoh Penggunaan:

  • Sistem antar lembaga pendidikan
  • Kerja sama antar instansi pemerintahan
  • Cloud antar rumah sakit
Community Cloud menjadi solusi tengah antara Public dan Private Cloud.

5. Hybrid Cloud

Hybrid Cloud adalah kombinasi dari dua atau lebih model deployment cloud, biasanya Public Cloud dan Private Cloud, yang saling terintegrasi.

Karakteristik Hybrid Cloud:

  • Fleksibilitas tinggi
  • Dapat memisahkan data sensitif dan non-sensitif
  • Efisiensi biaya dan keamanan seimbang

Contoh Penggunaan:

  • Aplikasi e-commerce
  • Sistem perusahaan besar
  • Manajemen data skala besar
Hybrid Cloud adalah model deployment yang paling banyak digunakan saat ini.

6. Perbandingan Model Deployment Cloud

  • Public Cloud: Murah, fleksibel, keamanan standar
  • Private Cloud: Aman, mahal, kontrol penuh
  • Community Cloud: Kolaboratif, biaya bersama
  • Hybrid Cloud: Fleksibel, paling kompleks
Pemilihan model deployment harus disesuaikan dengan kebutuhan organisasi, anggaran, dan tingkat keamanan data.
